# v1.3.0
- 加入 Content-Security-Policy、X-Content-Type-Options、X-Frame-Options 等 HTTP 安全头部，覆盖静态资源响应以强化纵深防御。
- 更新 deploy.sh 使用 jq 安全写入站点标题，并新增 HTTPS 部署提示与缺少 jq 的告警。
- 将前端保存密钥改为默认使用 localStorage 持久化，自动回退至 sessionStorage 并记录存储异常日志。

# v1.2.0
- 重构前端脚本为 ES 模块结构，新增 js/main.js、js/ui.js、js/api.js、js/settings.js 以拆分入口、渲染、数据请求与设置逻辑。
- 更新 index.html 使用模块入口脚本，确保浏览器以模块模式加载前端逻辑。
- 优化错误渲染逻辑，改用 DOM API 构建提示节点，避免直接插入 HTML 字符串。

# v1.1.0
- 移除 main.go 中的 Access-Control-Allow-Origin 响应头，避免过度开放的跨域策略。
- 在 script.js 的 createLinkCard 中校验链接协议，阻止注入 javascript: 等恶意 URL。
- 在 Dockerfile 中复制默认配置文件以配合 entrypoint.sh 的回退逻辑，确保容器首次启动可加载示例数据。

# v1.0.0
- 在 main.go 中强制要求配置 SAVE_TOKEN 并对缺失与错误令牌给出错误响应，防止未经授权的配置保存。
- 在 script.js 中将基于用户配置的内容渲染改为使用 textContent 与 DOM API，避免注入 HTML 引发的存储型 XSS。
